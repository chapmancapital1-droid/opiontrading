4. STRATEGIES (filter chips вҖ” all OptionScope IDs)
--------------------------------------------------
Credit income:
  bull_put_credit, bear_call_credit, iron_condor
Stock + option:
  cash_secured_put, covered_call
Debit verticals:
  bull_call_debit, bear_put_debit
Directional:
  long_call, long_put
Volatility:
  long_straddle, long_strangle (if cataloged)
Special:
  money_press_put_diagonal

Each strategy has:
  - thesis (bullish / bearish / neutral / vol)
  - preferred IV (elevated for credit, low/neutral for debit)
  - preferred trend
  - seed-friendly flag (defined risk preferred under $20k path)

5. FILTER DEFAULTS (user-editable)
----------------------------------
minPrice: 8
maxPrice: 150
maxResults: 15  (never invent fillers вҖ” return fewer if needed)
bias: any | bullish | bearish | neutral
liquidity: normal+ (reject illiquid when live chain known)
excludeEarnings: true (soft)
minFitScore: 0.45 (0вҖ“1)
useRhHistory: true (soft boost/penalty from personal lessons)
seedMode: true when equity < 20000 вҶ’ prefer defined-risk strategies

6. SCORING MODEL (engineering)
------------------------------
For each candidate ticker in the liquid universe:
  1. Hard filters: price вҲҲ [min,max], listed, not halted
  2. Base fit from strategy thesis vs synthetic/live context:
       - IV rank proxy, spot trend proxy, liquidity
  3. Strategy-specific bonuses (credit + elevated IV, debit + trend, etc.)
  4. RH history soft coach (rhHistoryScoreAdjust)
  5. Empire seed: boost defined-risk if seedMode
  6. Sort by fitScore desc, take top maxResults
  7. If < maxResults, return what qualifies (honest count)

Output row:
  { rank, symbol, spot, fitScore, fitPct, thesis, reasons[],
    strategyId, riskLabel, ivRankProxy, trendProxy, actions }
